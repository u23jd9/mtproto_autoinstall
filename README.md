- ✅ **Полностью автоматическая установка** MTProxy из исходников

## 🚀 Быстрый старт

### Установка одной командой

```bash
sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/ВАШ_АККАУНТ/mtproxy-installer/main/install.sh)"
```

Или через wget:
```bash
sudo bash -c "$(wget -qO- https://raw.githubusercontent.com/ВАШ_АККАУНТ/mtproxy-installer/main/install.sh)"
```

### Минимальные требования
- Ubuntu 18.04+ / Debian 9+
- 256 MB RAM (рекомендуется 512 MB)
- Свободный порт 8443 (или любой другой, можно изменить в скрипте)
- Root-доступ

### 1️⃣ Установка прокси
При выборе этого пункта скрипт:
- Запросит публичный и приватный IP
- Выполнит полную очистку предыдущих установок
- Установит все зависимости
- Скомпилирует MTProxy
- Сгенерирует секретный ключ
- Настроит systemd сервис
- Откроет порты
- Запустит прокси
- Сохранит ссылки для подключения в `/opt/MTProxy/links.txt`

### 2️⃣ Статус и ссылки
Показывает:
- Текущий статус сервиса (активен/неактивен)
- Последние логи
- Все сгенерированные ссылки для подключения

### 3️⃣ Добавление объявления
Позволяет добавить рекламный текст, который будет показываться пользователям прокси.

### 4️⃣ Диагностика
Проверяет:
- Открытые порты
- Статус сервиса
- Логи ошибок
- Наличие конфигурационных файлов

### 5️⃣ Полная деинсталляция
Удаляет:
- systemd сервис
- Все файлы прокси
- Пользователя mtproxy
- Правила firewall
- Cron задачи

## ⚠️ Возможные проблемы и их решение

### Проблема 1: Ошибка "cannot translate host"
**Симптом:** В логах ошибка `cannot translate host 'IP:PORT' in --nat-info`

**Решение:**
- При установке укажите правильные IP адреса
- Публичный IP должен быть доступен из интернета
- Приватный IP - внутренний IP сервера (можно узнать командой `ip a`)

### Проблема 2: Сервис не запускается (exit-code)
**Симптом:** `systemctl status MTProxy` показывает failed

**Решение:**
1. Запустите диагностику (пункт 4 в меню)
2. Проверьте порты: `ss -tlnp | grep 8443`
3. Проверьте логи: `journalctl -u MTProxy -n 50`
4. Убедитесь, что секретный ключ сгенерирован

### Проблема 3: Нет соединения через прокси
**Симптом:** Telegram пишет "Connecting..." или "Proxy unavailable"

**Решение:**
- Проверьте, открыт ли порт в firewall: `iptables -L -n | grep 8443`
- Проверьте, слушает ли процесс порт: `netstat -tlnp | grep 8443`
- Убедитесь, что публичный IP указан правильно
- Проверьте, не блокирует ли провайдер порт 8443

### Проблема 4: AWS Lightsail специфика
**Симптом:** Прокси работает, но нет соединения

**Решение:**
- При установке укажите:
  - Публичный IP: внешний IP инстанса
  - Приватный IP: IP из метаданных (обычно начинается на 172. или 10.)
- Убедитесь, что в firewall AWS открыты порты 8443 и 8888

## 🔧 Технические детали

### Структура файлов после установки
```
/opt/MTProxy/
├── mtproto-proxy          # Исполняемый файл
├── proxy-secret           # Секретный ключ Telegram
├── proxy-multi.conf       # Конфигурация Telegram
├── links.txt              # Сгенерированные ссылки
└── (файлы объявлений)     # При добавлении рекламы
```

### systemd сервис
```ini
[Unit]
Description=MTProxy
After=network.target

[Service]
Type=simple
User=mtproxy
Group=mtproxy
WorkingDirectory=/opt/MTProxy
ExecStart=/opt/MTProxy/mtproto-proxy -u mtproxy -p 8888 -H 8443 -S SECRET --aes-pwd proxy-secret proxy-multi.conf -M 1 --http-stats [--nat-info]
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### Используемые порты
| Порт | Назначение |
|------|------------|
| 8443 | Основной порт прокси |
| 8888 | Порт статистики (http://IP:8888/stats) |

## ❓ FAQ

### Можно ли изменить порт прокси?
Да, откройте скрипт и измените переменные `PROXY_PORT` и `STATS_PORT` в начале файла.

### Как обновить прокси до новой версии?
Просто запустите установку заново (пункт 1). Скрипт выполнит очистку и установит свежую версию.

### Безопасно ли использовать этот прокси?
Да, MTProxy только перенаправляет зашифрованный трафик Telegram и не может его расшифровать.

### Сколько пользователей выдержит прокси?
На сервере с 512 MB RAM и 1 vCPU прокси может обслуживать несколько сотен пользователей.

### Как проверить статистику?
Перейдите по адресу `http://ВАШ_IP:8888/stats` в браузере.

### Нужно ли обновлять конфигурацию вручную?
Нет, скрипт автоматически настраивает cron для ежедневного обновления в 4 утра.

### Что делать, если забыл ссылки для подключения?
Они всегда доступны в файле `/opt/MTProxy/links.txt` или в меню (пункт 2).

### Поддерживает ли скрипт IPv6?
Текущая версия сфокусирована на IPv4 для максимальной стабильности. IPv6 можно добавить при необходимости.
